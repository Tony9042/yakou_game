# 夜行 YAKOU — 遊戲企劃文件（延伸版）

> 定位：ROGUE-lite 動作 RPG，都會奇幻／霓虹美術，以「魂魄」系統貫穿戰鬥、成長與敘事。

---

## 0. 設計核心命題

過去 Roguelike 常見的問題：重複跑關容易失去敘事重量、meta 成長容易變成單純數值膨脹、多人容易破壞單機敘事節奏。

YAKOU 的解法：**魂魄系統同時是戰鬥資源、成長貨幣、與敘事鑰匙**。玩家在戰鬥中對每個付喪神做的「鎮壓／收容」選擇，會同時影響本輪戰力、長期養成、以及能否解鎖該角色的身世劇情。三個系統共用同一個決策點，而不是各自獨立的循環。

---

## 1. 核心遊戲循環（ROGUE）

```
橫丁（日間安全屋 / 社交大廳）
   │
   ▼
選擇今夜路線（節點式地圖，程序生成）
   │
   ├─ 遭遇（一般戰鬥）
   ├─ 黑市（花殘留魂魄／戰利品購買道具）
   ├─ 目擊（劇情碎片，通常無戰鬥）
   ├─ 契約（高風險高回報的精英戰）
   └─ 深夜案件（本夜 Boss）
   │
   ▼
黎明撤離 或 戰死
   │
   ▼
返回橫丁 → 結算殘留魂魄 → 天賦／解鎖／劇情
```

**地圖生成**：以模組化街區拼接（商店街、高架橋下、地下道、廢棄公寓、頂樓天台），確保每夜路線不同，但美術與節奏維持一致的「巷弄感」。

**死亡機制**：戰死會損失本輪未轉換的戰鬥用魂魄，但已收容並結算的殘留魂魄與天賦點永久保留——輸掉一夜不等於歸零，維持長線動機。敘事上，「戰死」是本夜依代（軀殼）毀去、主角之魂退回橫丁（見 §3.4），而非主角本體死亡，因此迴圈有劇情解釋而非單純失敗重來。

## 2. 魂魄系統（核心資源）

### 2.1 擊敗付喪神時的選擇

| 選項 | 效果 | 敘事意義 |
|---|---|---|
| **鎮壓** | 立即轉為本輪戰鬥增益（類似「祝福」，效果強但只在本輪有效） | 呼應「消滅」，較粗暴的解法 |
| **收容** | 納入「魂魄囊」，成為可切換隨行被動，佔用攜行容量 | 呼應世界觀「牠們只是太久沒被溫柔對待」，較溫柔的解法 |

攜行容量有限，收容越多可切換的被動越多，但能攜帶的一般道具越少——形成資源張力，而非單純「收集越多越好」。

### 2.2 殘留魂魄（Meta 貨幣）

跑完一夜（無論輸贏），所有本輪收容過的付喪神會依收容品質轉換為「殘留魂魄」，可在橫丁：

1. 永久解鎖該付喪神成為未來可召喚的常駐夥伴
2. 投入天賦樹
3. 解鎖該付喪神專屬的身世小劇情

流派差異化建議：「羈絆」流派（見 §3，導師為雫）直接強化收容效率／殘留魂魄產出，天生適合作為新玩家的系統教學起手式。

## 3. 角色客製與流派系統

### 3.0 設計取捨：客製主角 vs 固定角色

YAKOU 採 **「單一可客製主角 ＋ 可混搭流派」**，而非數名固定職業角色。理由是這款遊戲的核心是 Roguelite 的 build 續玩循環，而**魂魄收容本身已經是一條 build 客製軸**；若再把天賦鎖死在固定角色上，會出現兩個問題：

- **限縮組合爆炸**：固定四角色＝頂多四種戰鬥幻想；可混搭流派＝數十種本輪身分，才對得起類型核心。
- **削弱殘影敘事**：非同步殘影（§4.2）要傳達「這座城市住著別人的痕跡」，一個**玩家親手客製的 build ＋造型**遠比一隻標準角色更有重量。

**代價與對策**：客製化會削弱多人職業辨識度與「主玩某角色」的實況文化。對策是——① 流派帶隊伍光環做軟性分工，而非硬鎖職業（見 §4.1）；② 保留四名角色作為橫丁的**導師 NPC 與劇情人物**，身世劇情、命名權、實況梗全部延用；③ 提供以四人命名的**預設流派模板**作為新手起手式，想深玩再自由混搭。

### 3.1 客製主角

玩家擁有一名**可客製的主角**：外觀（白天造型、夜行裝束、隨行魂魄外顯）作為個人識別與展示層（呼應 §4.3 造型經濟、§8.4 VTuber 共生）。主角本身無固定職業，戰鬥身分由「投入的流派 × 收容的付喪神 × 本輪祝福」共同決定。

### 3.2 流派（魂之型）

四棵流派樹**不再綁定角色**，任何主角皆可自由投點並**同時投多棵樹混搭**。不採線性樹，採菱形分支（雙路徑可交會），在橫丁用殘留魂魄投點，並可付費用魂魄重置，避免玩家因試錯而卡死一條線。每種流派由一名角色導師（見 §3.3）於橫丁傳授並承載其身世劇情。

#### 刀語（導師：蓮 / 太刀使い）
- **拔刀**：單體爆發、高暴擊
- **居合**：反擊架勢，格擋後觸發強化攻擊
- **残心**：擊殺目標後延遲觸發範圍傷害

#### 結界（導師：灯 / 祈禱師）
- **封印**：控場技能，可使敵人陷入無力狀態而不擊殺（呼應「安撫而非消滅」的世界觀）
- **加護**：隊伍護盾／持續治療，多人模式核心
- **視**：顯示隱藏節點與寶物位置，強化探索面向

#### 疾走（導師：迅 / 機動）
- **突進**：位移技能可連段銜接
- **尾氣**：移動路徑留下範圍傷害
- **極速**：減防換取指數級加速與爆發，高風險流派

#### 羈絆（導師：雫 / 支援）
- **馴養**：提升收容成功率與魂魄囊容量
- **修復**：多人模式下的復活與續航技能
- **拾荒**：戰利品與殘留魂魄產出提升

**混搭範例**：刀語×疾走＝高機動爆發近戰；結界×羈絆＝收容續航輔助；三線淺投＝萬用生存流。流派間的交會節點可設計成需同時投兩線才解鎖的複合技，鼓勵跨流派 build。

### 3.3 四名角色 → 導師與敘事人物

蓮、灯、迅、雫不再是可選職業，而是常駐橫丁的 NPC：各自傳授一種流派、擁有獨立身世劇情、承載命名權與實況梗。玩家與導師的關係深度（透過該流派投點、劇情解鎖）成為另一條敘事進度軸，讓「選流派」同時是「選要親近誰的故事」。

### 3.4 主角本體設定：魂與依代

為了讓「可客製 avatar」與「強主線敘事」不衝突，主角採**雙層結構**——玩家操作的不是一具固定肉身，而是**一縷魂**，每夜附於一具可客製的**依代（軀殼）**行動。這讓客製自由與敘事錨點分屬兩層，各取所需，且與魂魄世界觀天然咬合。

| 層 | 是什麼 | 統一 or 客製 | 承載 |
|---|---|---|---|
| **魂（本體）** | 玩家真正的「主角」——一縷夜行之魂 | **統一、固定** | 主線敘事、情感錨點 |
| **依代（軀殼）** | 魂每夜附身的可客製身體與裝束 | **可客製、可換** | 外觀、造型經濟、build 展示層 |

**魂（本體）——固定的敘事錨點**
- 可命名，但擁有**固定聲線、少量關鍵台詞、一個謎樣來歷**（為何夜行、為何記不得生前）——「有底子但仍是你的角色」，參照《隻狼》《Hades》王子的定位。
- 魂的身世之謎即**主線懸念的一部分**：可與 §5.3「全城付喪神現象增加的原因」交織——玩家這縷魂為何在此、與都市更新／舊城改造下「被迫遷移的物與人」有何關聯。
- 因為主角是魂而非肉身，**祂的成長與記憶跨輪保留**，天然承載 meta 進度感，讓輸贏都有敘事重量。

**依代（軀殼）——客製與展示層**
- 「換外觀」有了世界觀解釋：一縷無固定肉身的魂，本就可附於不同軀殼、著不同裝束——§8.4 的 VTuber 換皮因此是**設定內合理**，而非外掛商城皮。
- 依代對應 §3.1 客製主角與 §3.2 流派 build：本輪身分＝魂所選的軀殼 × 投入的流派 × 收容的付喪神。

**與既有系統的咬合（此設定順帶把數個機制敘事化）**
- **死亡迴圈（§1）**：軀殼在夜裡毀去、魂於黎明退回橫丁——「戰死／撤離」由此獲得劇情解釋，而非單純的關卡失敗。
- **殘影（§4.2）**：你的殘影＝一縷穿著某具軀殼、帶著某套 build 的魂留下的痕跡，更貼合「城市住著別人痕跡」。
- **主題呼應（§2）**：玩家自己也是一縷「需要被安放的魂」，與「收容而非鎮壓付喪神」形成鏡像——你如何對待牠們，呼應著你自己的處境。

## 4. 多人模式

### 4.1 合作夜襲（2–4 人）
共享同一節點地圖與 Boss 戰，戰利品／魂魄可採分配制或票選制。因主角採客製流派而非固定職業，多人分工靠**軟性引導**成形，而非硬鎖：

- **流派光環**：投入某流派達門檻時，對隊友施加對應增益（如結界系給隊伍護盾、羈絆系提升全隊魂魄產出），讓玩家自然朝互補方向配置，但不強制。
- **組隊面板提示**：開戰前顯示隊伍目前的流派傾向（輸出／控場／機動／續航覆蓋度），提醒但不阻擋，保留全客製自由。

如此既維持組隊策略的可讀性，又不犧牲 Roguelite 的 build 自由度（參考《雨中冒險 2》全客製仍能自然成形分工）。

### 4.2 非同步殘影
玩家死亡時留下的「殘影」（build 配置＋死因）可能出現在其他玩家的世界中，作為一次性可召喚援軍，或作為環境警示標記。不需強制即時連線，也能維持「這座城市住著別人痕跡」的氛圍。

### 4.3 橫丁作為社交大廳
白天造型（街頭服飾、外套、鞋款）作為展示與客製化層，適合作為非付費致勝的變現方向，同時貼合遊戲既有的街頭時尚美術基調。

## 5. 敘事結構

Roguelike 的敘事難題是「重複的關卡如何講一個有推進感的故事」，解法拆成三層：

1. **目擊節點**：中途可選的短劇情，通常承載某一隻付喪神的身世碎片——這是「為什麼要收容而非鎮壓」的核心情感誘因。
2. **橫丁對話**：每輪結束後解鎖，依「花費的殘留魂魄／已收容的特定付喪神／里程碑」觸發，讓輸贏都有敘事進度感。
3. **主線懸念**：全城付喪神現象增加的原因，可隱約指向都市更新／舊城改造讓「物」與「人」同時被迫遷移，呼應世界觀而不需要過度說教。

## 6. 霓虹作為機制，不只是美術

### 光域機制
- 亮處（霓虹招牌密集區）：吸引更多敵人（蛾撲火意象），但強化部分技能（如蓮的残心、遠程視野向天賦）
- 暗巷：可潛行繞過遭遇節點，但削弱灯的結界類技能

這讓地圖選路成為真正的戰術決策，而不只是視覺風格切換。

### 霓虹充能
破壞或吸收特定招牌道具取得的次要資源，用來驅動角色的終極技能，直接把城市美學綁進戰鬥資源迴圈。

## 8. 實況主友善設計（Streamer & Viewer Interaction）

### 8.1 設計原則
放送向的設計不該是外掛在遊戲上的投票視窗，而是直接借用既有的「魂魄選擇」「殘影」「黑市／契約」系統，讓觀眾參與感覺是世界觀本身的一部分。

### 8.2 具體機制

| 機制 | 說明 | 掛勾的既有系統 |
|---|---|---|
| **委託節點** | 觀眾用頻道點數在黑市／契約節點提交委託（指定敵人類型、追加詛咒或祝福），主播抽到委託時可選擇是否接受 | 黑市／契約 |
| **觀眾殘影** | 觀眾提交自己的簡易配置（造型＋流派組合），有機率成為本輪的召喚援軍客串演出 | 非同步殘影 |
| **命名權** | 頻道點數／訂閱兌換為新捕獲付喪神命名，形成跨場次的固定哏 | 魂魄收容 |
| **岔路投票** | 特定節點（黑市／目擊）開放聊天室投票決定分支，訂閱者可加權，避免灌票失控 | 節點地圖 |
| **死亡回顧演出** | 死亡時觸發簡短「戰績回顧」（本輪收容清單／擊殺數／存活時間），兼具安慰效果與剪輯素材 | 死亡結算 |
| **每日巷弄** | 固定種子的每日挑戰，方便主播與觀眾社群互相比較成績 | 節點生成 |
| **觀眾合力夜戰** | 每週限定 Boss，頻道點數池累積到門檻解鎖額外支援單位，作為固定「週末企劃」 | 深夜案件／多人 |

### 8.3 注意事項
- 投票／委託類機制務必保留「主播可否決」的空間，避免節奏完全被聊天室綁架而失去可玩性。
- 觀眾殘影的出現機率建議偏低，或做成可獨立開關的「直播模式」，避免每輪都出現而失去新鮮感。
- 命名權與投票權重最好綁定「訂閱／頻道點數」而非免費灌票，同時作為變現點且不影響遊戲平衡（不给數值優勢，只給敘事/展示層的參與感）。

### 8.4 VTuber／YouTuber 共生設計

客製主角這一步，讓 YAKOU 天生適合與 VTuber／內容創作者生態結合——**「玩家有自己的角色模型」與 VTuber 的核心（自己的皮）是同一件事**。遊戲畫面因此能成為創作者的第二個舞台，而非一款被實況的普通遊戲。

#### 設計原則
借用既有的客製化、造型經濟與殘影系統來承載創作者身分，避免做成外掛的聯名皮膚；讓「這是我的角色」自然延伸成「這是我的頻道形象」。

| 機制 | 說明 | 掛勾的既有系統 |
|---|---|---|
| **人設同步** | 主角客製面板提供貼近 VTuber 風格的部件（髮色、瞳色、獸耳／尾、發光紋樣），創作者可把 avatar 調成貼近自身人設，遊戲畫面即個人舞台 | 角色客製（§3.1）、造型經濟 |
| **簽名殘影** | 創作者的殘影可標註頻道名／連結標記（僅展示層，不給數值），出現在觀眾世界時形成「巡演」般的曝光 | 非同步殘影（§4.2） |
| **聯名付喪神** | 與特定創作者合作推出主題付喪神／造型，收容後可解鎖該創作者參與配音或撰寫的專屬身世小劇情 | 魂魄收容、目擊劇情 |
| **頻道橫丁** | 創作者可擁有可展示的個人化橫丁（擺設、招牌、常駐夥伴），作為對觀眾的固定門面與社群聚點 | 橫丁社交大廳（§4.3） |
| **剪輯友善輸出** | 死亡回顧與 build 總結產出直向（手機短影音）版面與可一鍵匯出的戰績卡，貼合 YouTube Shorts／直播剪輯工作流 | 死亡回顧演出（§8.2） |

#### 注意事項
- 人設部件應涵蓋足夠廣的風格（非單一萌系），避免綁死特定社群；核心部件免費、進階裝飾走造型經濟，維持非付費致勝。
- 聯名內容需保留「不影響平衡」原則：只在敘事／展示層給差異，數值中立。
- 頻道標記／連結僅在展示層呈現，避免變成遊戲內廣告噪音而傷害沉浸感；建議可由觀眾端自行開關。
