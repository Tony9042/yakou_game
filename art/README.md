# 美術

目前所有畫面由 `combat.gd` 等**程式生成**（向量圖形＋光暈＋陰影＋粒子），零外部素材。
以下是日後接入真美術（手繪／像素／AI 生成 sprite）的規劃，避免現在卡在等素材。

## 目前程式生成的內容（戰鬥場）

- **夜街天際線**：剪影大樓、亮窗、霓虹招牌（帶光暈）
- **地面**：柏油底色＋霓虹反光街道
- **角色／敵人**：向量拼合的人形與造型，加上**落地陰影**與**依代／類型色的輪廓光暈**
- **氛圍**：緩緩上升的魂魄微光粒子

光暈與陰影共用一張程式生成的柔和放射貼圖（`_soft_dot()`），因此換成真 sprite 後這些效果仍可沿用。

## 接入真 sprite 的建議路徑

當你有素材時，建議放到：

| 路徑 | 用途 |
|---|---|
| `res://art/vessels/<依代名>.png` | 角色（依代）立繪／sprite sheet |
| `res://art/enemies/<類型>.png` | 敵人（chaser/ranged/charger/boss） |
| `res://art/env/*.png` | 街景、招牌、地磚等場景元件 |

接法：在 `_build_player_visual()` / `_spawn_enemies()` 開頭判斷對應檔案是否存在
（`ResourceLoader.exists(path)`），存在就用 `Sprite2D`／`AnimatedSprite2D` 取代向量拼圖，
不存在則保留現有程式生成版——與音效的覆蓋機制同理。
（此判斷邏輯待有實際檔案可測時再加，以免無素材下空轉。）
